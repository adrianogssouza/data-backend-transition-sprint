# Sprint Overview

## Visão geral

Esta sprint foi estruturada como uma etapa prática da minha transição para as áreas de **Engenharia de Dados** e **Desenvolvimento Backend**.

O foco principal é sair do estudo passivo e consolidar uma base técnica aplicada por meio de projetos, documentação e prática orientada a dados.

---

## Objetivo principal

Construir uma base prática em:

- Python para backend e manipulação de dados
- SQL para análise, modelagem e performance
- FastAPI para desenvolvimento de APIs
- PostgreSQL para persistência e modelagem relacional
- dbt para fundamentos de analytics engineering
- Docker como parte do fluxo de entrega da aplicação

---

## Contexto da sprint

Durante este ciclo, a proposta é estudar com foco em execução e documentação, organizando a evolução técnica em um mono-repositório no GitHub e conectando essa jornada com minha presença profissional no LinkedIn e, futuramente, com um site de portfólio.

A sprint não foi pensada apenas como um período de estudo, mas como uma construção real de base técnica, narrativa profissional e ativos de portfólio.

---

## Período

- **Início:** 19/04/2026
- **Término planejado:** 06/05/2026

---

## Restrições do cronograma

- domingos com no máximo 2 horas de estudo
- 21/04 sem estudos
- foco em manter consistência sem transformar o processo em consumo passivo de conteúdo

---

## Estrutura de aprendizado

A sprint está organizada em três frentes principais:

### 1. Aplicação e backend
- Python
- POO
- SQLModel / SQLAlchemy
- Pydantic
- FastAPI
- CRUD integrado com banco

### 2. Banco, SQL e analytics engineering
- consultas SQL
- joins
- window functions
- CTEs
- subqueries
- views
- transações ACID
- índices
- particionamento
- dbt

### 3. Documentação e posicionamento profissional
- organização do mono-repositório
- documentação técnica
- registros de evolução
- estratégia de conteúdo para LinkedIn
- preparação futura para portfólio em site

---

## Projetos previstos

### Python CRUD API
Projeto principal da sprint, com foco em backend orientado a dados.

### SQL Analytics Lab
Laboratório de consultas e prática de SQL analítico e de performance.

### dbt Lab
Espaço de aprendizado voltado à transformação e organização da camada analítica.

---

## Prioridades da sprint

### Prioridade máxima
- consolidar Python até a construção da API CRUD
- fortalecer SQL até os tópicos de análise, modelagem e performance
- conectar backend com banco relacional

### Prioridade alta
- documentar o processo de forma clara
- estruturar o GitHub com qualidade
- gerar material reaproveitável para LinkedIn e portfólio

### Prioridade complementar
- ampliar a visão sobre Docker e demais componentes da stack
- preparar terreno para projetos futuros de maior complexidade

---

## Critérios de sucesso

Ao final da sprint, o resultado esperado é:

- repositório organizado e bem documentado
- projeto de API CRUD estruturado
- laboratório de SQL com exercícios e anotações relevantes
- base inicial em dbt documentada
- material suficiente para movimentar LinkedIn de forma estratégica
- fundação aproveitável para um futuro site de portfólio

---

## Forma de acompanhamento

O acompanhamento da sprint está dividido em três blocos:

- **E1** → estruturação e documentação do repositório
- **E2** → plano editorial do LinkedIn
- **E3** → modelo de documentação diária e recaps

O controle operacional pessoal está sendo mantido em arquivos locais privados, fora do versionamento público do GitHub.

---

## Próximo passo

Após este overview, os próximos passos são:

1. criar o template de registro diário
2. criar a documentação base dos projetos

## Rotina de execução

A execução da sprint seguirá um modelo simples e consistente:

### Fluxo diário
- Estudar (30–60 min)
- Aplicar (30–60 min)
- Registrar (10–15 min)
- Encerrar (5 min)

### Frequência de documentação
- Registro diário → todos os dias de estudo
- Recap semanal → 1x por semana
- Checkpoint técnico → a cada 3–4 dias

### Objetivo da rotina
Garantir consistência, aplicação prática e retenção do conhecimento sem tornar o processo pesado ou inviável dentro da rotina diária.
